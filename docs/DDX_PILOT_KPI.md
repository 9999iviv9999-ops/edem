# KPI для пилота DDX

## Что считается

Скрипт [`scripts/ddx-pilot-kpi-report.mjs`](../scripts/ddx-pilot-kpi-report.mjs) читает **боевую** (или staging) БД по `DATABASE_URL`.

- **DDX-залы:** `Gym.chainName` начинается с `DDX` — как в каталоге приложения.
- **Период:** календарные даты `--from` / `--to` (границы суток в **UTC**).
- Опционально **`--cities`:** фильтр по `User.city` (через запятую).

Метрики:

| Поле в отчёте | Смысл |
|---------------|--------|
| Регистрации | Новые пользователи за период, не забанены |
| Основной зал DDX в периоде | `UserGymMembership` с `isPrimary` и клубом DDX, `createdAt` в периоде |
| Лайки / матчи / сообщения | Активность, где контекстный зал — DDX |
| Топ клубов | По числу выборов «основной зал» DDX в периоде |

**Удержание (retention)** по когортам в скрипте не считается — при необходимости добавьте отдельный запрос или расширение скрипта.

## Запуск

```bash
set -a && [ -f .env ] && . ./.env && set +a   # или export DATABASE_URL=...

npm run report:ddx-pilot -- --from 2026-04-01 --to 2026-06-30

# только Москва и СПб (по городу в профиле)
npm run report:ddx-pilot -- --from 2026-04-01 --to 2026-06-30 --cities "Москва,Санкт-Петербург"

# машиночитаемый вывод
npm run report:ddx-pilot -- --from 2026-04-01 --to 2026-06-30 --json > kpi.json
```

На сервере можно выполнить из каталога репозитория с тем же `.env`, что у API, либо:

```bash
docker compose exec -T api node scripts/ddx-pilot-kpi-report.mjs --from 2026-04-01 --to 2026-06-30
```

(путь к скрипту внутри контейнера — `scripts/` в образе, если копируется весь проект.)

## Отчёт для DDX

1. Сгенерируйте таблицу (`npm run report:ddx-pilot …`).
2. Добавьте вручную блок **«Выводы»**: выполнение KPI пилота, города, что менять в продукте/маркетинге.
3. При необходимости приложите **скрин** или PDF из админки / графики за тот же период.
