# ЭДЕМ: Инструкция по админке

## 1) Как попасть в админку

1. Войти под пользователем с `isAdmin = true`.
2. Открыть сайт `https://edem.press`.
3. Перейти в раздел `Админка` в верхнем меню (или вручную открыть `/admin`).
4. Если доступа нет, система вернет на главную страницу.

## 2) Что есть в админке

На странице `/admin` доступны блоки:

- **Ключевые метрики**: регистрации за день, активные 24ч, открытые жалобы, всего пользователей.
- **Пользователи**: поиск по имени/email/телефону, бан/разбан.
- **Жалобы**: перевод статусов `open -> in_review -> resolved/dismissed`.
- **Каталог залов**: поиск и редактирование карточек залов.
- **Комментарии к анкетам**: удаление токсичных/спам-комментариев.
- **Аудит-лог**: кто, что и когда сделал в админке.

## 3) Основные рабочие сценарии

### A. Обработать жалобу на пользователя

1. Открыть блок `Жалобы`.
2. Найти жалобу со статусом `open`.
3. Нажать:
   - `В работу` — если начинаете разбор,
   - `Решено` — если вопрос закрыт,
   - `Отклонить` — если жалоба без оснований.
4. При необходимости перейти в `Пользователи` и выполнить бан/разбан.

### B. Заблокировать или разблокировать пользователя

1. Открыть блок `Пользователи`.
2. Найти пользователя через строку поиска.
3. Нажать:
   - `Заблокировать` — с причиной модерации,
   - `Разблокировать` — чтобы вернуть доступ.

> При бане активные refresh-сессии пользователя отзываются, доступ прекращается.

### C. Удалить нарушающий комментарий

1. Открыть блок `Комментарии к анкетам`.
2. Найти комментарий.
3. Нажать `Удалить комментарий`.
4. Действие попадет в `Аудит-лог`.

### D. Исправить данные зала

1. Открыть `Каталог залов`.
2. Найти нужный зал.
3. Обновить поля: `Название`, `Адрес`, `Город`, `Сеть`.
4. Нажать `Сохранить`.

## 4) Правила работы модератора

- Сначала ставить жалобу `В работу`, потом принимать решение.
- Банить только за конкретное нарушение, фиксируя причину.
- Не удалять контент без причины (ориентироваться на правила платформы).
- Проверять `Аудит-лог` в конце смены.

## 5) Быстрый чек в начале дня

1. Открыть `/admin`.
2. Проверить метрики и количество `open` жалоб.
3. Разобрать новые жалобы.
4. Проверить новые комментарии.
5. Проверить `Аудит-лог` на аномалии.

## 6) Что делать при ошибке доступа

- Убедиться, что у аккаунта `isAdmin = true`.
- Проверить, что access/refresh токены валидны (перелогиниться).
- Проверить доступность API: `GET /api/health`.
