# Чеклист безопасности ЭДЕМ (операционный)

Код уже усиливается в репозитории; ниже — что стоит сделать **вне** приложения и что периодически перепроверять.

## Обязательно в проде

- **HTTPS** везде, корректный **`NODE_ENV=production`** у API (HSTS и логи).
- **`JWT_SECRET`**, **`ADMIN_MODERATION_KEY`**, ключи S3/VK/Google — **длинные случайные**, не в git, ротация при компрометации.
- **`ACCESS_TOKEN_TTL_MINUTES`** — 30–120 (не оставлять `0` без веской причины).
- **`CORS_ALLOW_VERCEL_APP=0`**, явный список в **`CORS_ALLOW_ORIGINS`** для всех фронтов с `credentials`.
- **Бэкапы БД**, проверка восстановления, шифрование хранилища бэкапов.

## Инфраструктура

- **Nginx / WAF**: лимиты `client_max_body_size`, таймауты, при необходимости rate limit на уровне прокси.
- Несколько инстансов API: вынести **rate limit** и **oauth exchange** в **Redis** (сейчас в памяти процесса).
- Мониторинг **5xx**, аномального трафика на `/api/auth/*`, `/health`.

## Зависимости и аудит

- Регулярно: `npm audit`, обновления патчей, Dependabot / аналог.
- Раз в год или при крупных изменениях: **внешний пентест** или баг-баунти.

## Продукт и комплаенс

- Политика конфиденциальности и оферта **актуальны** относительно реальных потоков данных.
- Юридическая оценка **хранения фото**, возраста, геоданных (залы/город).

## После инцидента

- Смена **`JWT_SECRET`** (разлогин всех), проверка **audit log** модерации, утечки в логах прокси.
